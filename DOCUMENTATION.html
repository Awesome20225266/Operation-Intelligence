üìä Mathematical Formulae & KPIs

‚ö° Core Performance Metrics

Performance Ratio (PR%)
PR(%) = (Actual Energy / Expected Energy) √ó 100

Expected Energy = POA √ó Array Capacity √ó Œ∑_system

Source: daily_kpi.pr_percent (directly from Excel)
Threshold: Typical PR > 75% is considered healthy

Capacity Utilization Factor (CUF%)
CUF(%) = (Actual Energy / (Rated Capacity √ó 24h)) √ó 100

AC CUF = inv_gen_kwh / (operational_ac_mw √ó 1000 √ó 24)
DC CUF = (similar, using operational_dc_mwp)

Source: daily_kpi.ac_cuf_percent, daily_kpi.dc_cuf_percent

Plant Availability (PA%)
PA(%) = (Available Hours / Total Hours) √ó 100

Available Hours = 24h - Maintenance Time - Fault Time

Source: daily_kpi.pa_percent

Energy Generation Availability (EGA%)
EGA(%) = (Energy Available / Energy Expected) √ó 100

Source: daily_kpi.ga_percent

üí∞ Revenue Calculation

Daily Revenue (INR)
Revenue = inv_gen_kwh √ó Tariff (INR/kWh)

Example: 50,000 kWh √ó ‚Çπ6.00/kWh = ‚Çπ3,00,000

Default Tariff: ‚Çπ6.00/kWh (configurable in UI)

üìâ Loss Calculations

Energy Gap (kWh)
Gap = Target Energy - Actual Energy
Gap = b_energy_kwh - inv_gen_kwh
Soiling Loss (kWh)
Soiling Loss = (b_sl_percent / 100) √ó b_energy_kwh

Source: budget_kpi.b_sl_percent (Budg Soiling Loss %)
Note: Unattributed gap = Total Gap - Soiling Loss

üîß Equipment-Level Metrics

Inverter Performance Ratio (per equipment)
INV_PR(%) = pr.pr_percent

Query: SELECT equipment_name, pr_percent FROM pr
WHERE site_name = ? AND date = ?
String Yield Deviation (SYD%)
SYD(%) = syd.syd_percent

Query: SELECT equipment_name, syd_percent FROM syd
WHERE site_name = ? AND date = ?

Interpretation: Lower SYD% = better performance (closer to expected yield)

SCB Current Deviation Detection
For each timestamp (t), each SCB (i):
avg_i = mean(current_i across all timestamps in period)
std_i = stddev(current_i)

Deviation Score = (current_i[t] - avg_i) / std_i

Flag if: Deviation Score < threshold (e.g., -3.0 sigma)

Use Case: Raw Analyser & SCB OT modules detect underperforming strings

üìà Time-Series Approximations

Dashboard shows "last 24h" charts but DuckDB stores daily totals. We approximate hourly profiles using bell curves.
# Solar Profile Generation (Gaussian bell curve) def _solar_profile_24h(total_kwh, sunrise_hour=6, sunset_hour=18): hours = np.arange(24) mask = (hours >= sunrise_hour) & (hours <= sunset_hour) x = np.linspace(-2.5, 2.5, mask.sum()) bell = np.exp(-(x**2)) # Gaussian curve y = np.zeros(24) y[mask] = bell / bell.sum() * total_kwh # Normalize to sum = total return y # Example: 50,000 kWh distributed over 6am-6pm with peak at noon profile = _solar_profile_24h(50000) # [0, 0, 0, 0, 0, 0, 1200, 3500, 5800, 7200, 8000, 7500, 5500, 3200, 1000, 0, ...]

üìú Business Rules & Regulations

üîí Data Integrity Rules

Rule #1: Missing Data ‚Üí Zero
All missing, blank, non-numeric, or error values are coerced to 0.0 during ingestion. This prevents calculation errors and ensures consistent reporting.
Rule #2: Full-Row Duplicate Detection
A row is considered a duplicate ONLY if ALL column values match an existing row exactly. Rows that share a primary key but differ in any metric are updated via MERGE (not skipped).
Rule #3: 60-Second Decision Window
When exact duplicates are detected, the loader prompts: "Type 'replace all' within 60 seconds to overwrite exact duplicates."
‚Ä¢ Timeout/no input ‚Üí Skip duplicates, load only new/changed rows
‚Ä¢ "replace all" ‚Üí Delete existing, re-insert all
Rule #4: Site Name Extraction
Excel filename DGR_SiteName.xlsx ‚Üí site_name = SiteName
If filename doesn't start with "DGR_", use entire filename stem.
Rule #5: Design Data Table Lock
Only plant_details and array_details tables can be created/updated by design injestor. Attempts to write other tables will raise ValueError.

üéØ Calculation Regulations

Regulation: PR Threshold
Performance Ratio (PR%) must be ‚â• 75% to be considered "healthy"
Lower values trigger investigation in Operation Theatre
Regulation: SCB Deviation Threshold
Deviation Score = (Current - Mean) / StdDev
Flag SCB if: Deviation Score < -3.0 (3 sigma below mean)

This threshold is configurable in SCB OT module
Regulation: Real-Time Power Approximation
RT Power (kW) = Daily Energy (kWh) / 24h

Note: This is an approximation since DB stores daily totals, not instantaneous power
